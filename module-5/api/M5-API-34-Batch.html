Lernziele

Konzept — Wann Batch
💡 Batch ist asynchron: Du schickst hunderte oder tausende Anfragen auf einmal, gehst weg, und holst die Ergebnisse später ab (bis zu 24h). Dafür gibt es 50% Rabatt. Perfekt, wenn niemand vor dem Bildschirm auf eine sofortige Antwort wartet.
Batch erstellen
from anthropic import Anthropic
client = Anthropic()

batch = client.messages.batches.create(
  requests=[
    {
      "custom_id": "frage-1", # deine ID zum Wiederfinden
      "params": {
        "model": "claude-sonnet-4-5",
        "max_tokens": 512,
        "messages": [{"role": "user", "content": "Fasse Text A zusammen."}],
      },
    },
    {
      "custom_id": "frage-2",
      "params": {
        "model": "claude-sonnet-4-5",
        "max_tokens": 512,
        "messages": [{"role": "user", "content": "Fasse Text B zusammen."}],
      },
    },
  ]
)
print(batch.id, batch.processing_status) # z.B. "in_progress"
✅ custom_id ist Pflicht: Die Reihenfolge der Ergebnisse ist NICHT garantiert. Über deine selbstvergebene custom_id ordnest du jede Antwort wieder der richtigen Eingabe zu.
Status pollen & Ergebnisse abholen
import time

# Solange pollen, bis fertig
while True:
  b = client.messages.batches.retrieve(batch.id)
  if b.processing_status == "ended":
    break
  time.sleep(30) # nicht zu oft pollen

# Ergebnisse streamen und per custom_id zuordnen
for result in client.messages.batches.results(batch.id):
  if result.result.type == "succeeded":
    msg = result.result.message
    print(result.custom_id, msg.content[0].text)
  else:
    print(result.custom_id, "FEHLER", result.result.type)
🚫 Falle: Batch ist KEIN Echtzeit-Tool. Wenn ein User auf dem Bildschirm wartet, ist Batch falsch — nutz dafür einen normalen messages.create-Call. Batch ist für Hintergrund-Jobs.

Szenario — 1000 Reviews klassifizieren

Szenario: Nächtliches Stimmungs-Tagging

1
Du baust 1000 Requests, jeder mit eigener custom_id (= Review-ID)
2
batches.create(requests=[...]) abschicken — Status "in_progress"
3
Alle 30 Sek retrieve, bis processing_status == "ended"
4
results() durchlaufen, jede Antwort über custom_id der Review zuordnen
Der Spareffekt
Dieselben 1000 Anfragen kosten als Batch nur die Hälfte. Bei großen Mengen ist das der Unterschied zwischen rentabel und unbezahlbar.
💡 Jetzt du: Bau einen kleinen Batch mit 3 Requests (je eigener custom_id). Schick ihn ab, poll den Status, hol die Ergebnisse und gib sie nach custom_id sortiert aus.
