Setup — SDK & Client
💡 Ein Endpunkt für alles: Chat, Tool Use, Vision, Caching, Streaming — alles läuft über client.messages.create(). Du lernst hier die Basis, auf der jede weitere Lektion aufbaut.
# Installation (einmalig)
pip install anthropic

# Der API-Key wird aus der Env-Variable ANTHROPIC_API_KEY gelesen.
# export ANTHROPIC_API_KEY="sk-ant-..." (niemals hardcoden!)
from anthropic import Anthropic

client = Anthropic() # liest ANTHROPIC_API_KEY aus der Env

response = client.messages.create(
  model="claude-sonnet-4-5",
  max_tokens=1024,
  messages=[
    {"role": "user", "content": "Erklär mir die Messages API in einem Satz."}
  ],
)

print(response.content[0].text)
print(response.usage.input_tokens, response.usage.output_tokens)
✅ Antwort auslesen: Der Text steht in response.content[0].text. Die Token-Zahlen liegen in response.usage.input_tokens und .output_tokens — daraus berechnest du deine Kosten.

🚫 Häufige Falle: Der System-Prompt gehört NICHT in das messages-Array. Er ist ein eigener Parameter system="...". Ein {"role":"system"}-Eintrag wie bei anderen APIs funktioniert hier nicht.
response = client.messages.create(
  model="claude-sonnet-4-5",
  max_tokens=1024,
  system="Du bist ein knapper deutscher Tech-Coach. Antworte in maximal 3 Sätzen.",
  messages=[
    {"role": "user", "content": "Was ist ein Token?"}
  ],
)
Multi-Turn — Kontext aufbauen
💡 Die API ist zustandslos: Sie merkt sich nichts. Für ein Gespräch hängst du die vorherige Antwort als role: "assistant" selbst ans Array an und schickst alles erneut mit.
messages = [
  {"role": "user", "content": "Nenn mir eine Geschäftsidee mit KI."}
]
r1 = client.messages.create(model="claude-sonnet-4-5", max_tokens=512, messages=messages)

# Antwort von Claude ans Gespräch anhängen
messages.append({"role": "assistant", "content": r1.content[0].text})
# Neue Folgefrage stellen
messages.append({"role": "user", "content": "Und wie verdiene ich damit Geld?"})

r2 = client.messages.create(model="claude-sonnet-4-5", max_tokens=512, messages=messages)
print(r2.content[0].text)
✅ Regel: Rollen wechseln sich immer ab — user, assistant, user, … Mit jedem Turn wächst der Input und damit die Kosten. Hier setzt später das Prompt Caching (L32) an.
Szenario — Vom ersten Call zum Gespräch

Szenario: Ein kleiner Chat-Bot in 4 Schritten

1
Client erstellen, Key aus der Env — client = Anthropic()
2
Verhalten über system festlegen (z. B. "Du bist ein Support-Bot")
3
Erste Frage als role: "user" senden, Antwort aus content[0].text lesen
4
Antwort als assistant anhängen, neue Frage anhängen, erneut senden
Kernidee
Du verwaltest den Verlauf selbst. Das messages-Array IST das Gedächtnis deiner App.
💡 Jetzt du: Schreib ein Skript first_call.py mit einem system-Prompt deiner Wahl. Setz einen ersten Call ab, gib response.content[0].text aus und logge danach input_tokens + output_tokens. Erweitere es dann zu zwei Turns.
