Lernziele

  • Eine Datei hochladen und die zurückgegebene file_id erhalten
  • Die Datei per file-Block in einer Nachricht referenzieren (mit Beta-Header)
  • Verstehen, wann Files besser ist als base64-inline
Konzept — Upload einmal, Referenz oft
💡 Das Problem mit base64: Du musst dieselbe große Datei bei JEDEM Call neu mitschicken. Mit der Files API lädst du sie EINMAL hoch, bekommst eine file_id und referenzierst diese danach beliebig oft — schlank und wiederverwendbar.
Datei hochladen
from anthropic import Anthropic
client = Anthropic()

uploaded = client.beta.files.upload(
  file=("report.pdf", open("report.pdf", "rb"), "application/pdf"),
)
print(uploaded.id) # z.B. "file_abc123" -> das ist deine file_id
🚫 Beta-Header nicht vergessen: Die Files API ist als Beta verfügbar. Du musst den Beta-Header mitschicken — entweder über client.beta... wie hier, oder per extra_headers={"anthropic-beta": "files-api-..."}. Ohne ihn lehnt die API den Call ab.
Datei in einer Nachricht referenzieren
resp = client.beta.messages.create(
  model="claude-sonnet-4-5", max_tokens=1024,
  messages=[{
    "role": "user",
    "content": [
      {
        "type": "document",
        "source": {"type": "file", "file_id": uploaded.id},
      },
      {"type": "text", "text": "Fasse die wichtigsten 3 Punkte zusammen."},
    ],
  }],
)
print(resp.content[0].text)
✅ Wiederverwenden: Dieselbe file_id kannst du in beliebig vielen weiteren Calls nutzen — etwa für zehn verschiedene Fragen zum selben PDF, ohne es zehnmal hochzuladen.

Wann Files, wann inline

Files nutzen

  • große PDFs / Datasets
  • mehrfach befragt
  • über mehrere Sessions

Egal

  • mittlere Dateien
  • 2-3 Fragen
  • beides ok

  • kleines Bild einmalig
  • schneller Test
  • kein Wiederverwenden

Der Gewinn
Statt das PDF zehnmal als base64 zu kodieren und zu übertragen, lädst du es einmal hoch und sparst Bandbreite, Code und Zeit.
💡 Jetzt du: Lad eine kleine PDF- oder Textdatei hoch, merk dir die file_id und stell zwei verschiedene Fragen dazu — beide mit derselben file_id, ohne erneuten Upload. Vergiss den Beta-Header nicht.
