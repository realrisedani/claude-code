Lernziele

  • Ein Tool mit Name, Description und input_schema (JSON-Schema) definieren
  • Den Tool-Use-Loop verstehen: Claude fragt an, du führst aus, du gibst tool_result zurück
  • Eine echte Python-Funktion mit Claude verdrahten
Konzept — Was Tool Use löst
💡 Das Modell rechnet nicht, ruft nicht ab, weiß nichts Aktuelles. Mit Tool Use gibst du Claude Funktionen an die Hand: Wetter abfragen, Datenbank lesen, E-Mail senden. Claude entscheidet WANN und MIT WELCHEN Argumenten — du führst aus.

1 · Du definierst

  • Name + Description
  • input_schema (welche Args)
  • Liste an tools=[]

2 · Claude wählt

  • liefert stop_reason="tool_use"
  • tool_use-Block mit Args
  • du führst die Funktion aus

3 · Du antwortest

  • tool_result zurücksenden
  • mit tool_use_id verknüpft
  • Claude formuliert finale Antwort
Tool definieren
from anthropic import Anthropic
client = Anthropic()

tools = [{
  "name": "get_weather",
  "description": "Gibt das aktuelle Wetter in Grad Celsius für eine Stadt zurück.",
  "input_schema": {
    "type": "object",
    "properties": {
      "city": {"type": "string", "description": "Name der Stadt"}
    },
    "required": ["city"]
  }
}]
🚫 Description ist alles: "Gibt Wetter zurück" ist zu schwach. "Gibt das aktuelle Wetter in °C für eine Stadt zurück, wenn der User nach Wetter fragt" sagt Claude WANN und WIE es zu nutzen ist. Schlechte Beschreibung = falsche oder fehlende Tool-Calls.
Der Tool-Use-Loop
def get_weather(city):
  # In echt: API-Aufruf. Hier ein fixer Wert.
  return "18°C, leicht bewölkt"

messages = [{"role": "user", "content": "Wie ist das Wetter in Berlin?"}]

resp = client.messages.create(
  model="claude-sonnet-4-5", max_tokens=1024,
  tools=tools, messages=messages,
)

if resp.stop_reason == "tool_use":
  tool = [b for b in resp.content if b.type == "tool_use"][0]
  result = get_weather(**tool.input) # Funktion ausführen

  # Claudes Tool-Anfrage + dein Ergebnis ans Gespräch hängen
  messages.append({"role": "assistant", "content": resp.content})
  messages.append({"role": "user", "content": [{
    "type": "tool_result",
    "tool_use_id": tool.id,
    "content": result,
  }]})

  final = client.messages.create(
    model="claude-sonnet-4-5", max_tokens=1024,
    tools=tools, messages=messages,
  )
  print(final.content[0].text) # "In Berlin sind es 18°C..."
✅ Wichtig: Das tool_result wird als role: "user" gesendet und über die tool_use_id mit der Anfrage verknüpft. Erst dieser zweite Call liefert die fertige Textantwort.

Szenario — Wetter-Bot

Szenario: User fragt nach dem Wetter

1
User: "Wie ist das Wetter in Berlin?" → Call mit tools=[get_weather]
2
Claude liefert stop_reason="tool_use" + input={"city":"Berlin"}
3
Du rufst get_weather("Berlin") auf → "18°C, leicht bewölkt"
4
Du sendest tool_result zurück → Claude: "In Berlin sind es 18°C, leicht bewölkt."
Der Loop
Claude kann mehrere Tools hintereinander aufrufen. Solange stop_reason == "tool_use", wiederholst du den Schritt — bis Claude eine Textantwort liefert.
💡 Jetzt du: Definier ein eigenes Tool, z. B. add(a, b) mit input_schema für zwei Zahlen. Frag Claude "Was ist 47 plus 58?", führ die Funktion aus und schick das tool_result zurück. Prüf, ob die finale Antwort 105 enthält.
