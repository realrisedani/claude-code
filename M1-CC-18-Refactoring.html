Safety-Checklist

  • git status clean?
  • Tests grün vor Start?
  • Scope klar definiert?
  • Worktree für großen Refactor?

5 Patterns

  • Rename: Klarheit herstellen
  • Extract: Cohesion erhöhen
  • Split File: >300 Zeilen aufteilen
  • Modernize + Migration

Migration-Disziplin

  • Beide Systeme parallel laufen
  • Neues für neue Features nutzen
  • Altes schrittweise migrieren
  • Altes System löschen

Commit-Style

  • refactor(scope): kurze Beschreibung
  • Atomar — ein Pattern pro Commit
  • Tests grün vor jedem Commit
  • CI muss durchlaufen
⚠ Rename-Falle: Nicht nur Code-Vorkommen — auch Strings, Kommentare, Docs und Config-Files prüfen!
