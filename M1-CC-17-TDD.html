TDD-Workflow

  • "Schreib Tests für X OHNE Implementation"
  • Tests reviewen — vollständig?
  • Tests rot? → "Implementiere bis grün"
  • Dann Refactor ohne Angst

Warum mit Claude perfekt

  • Claude liebt klare Specs
  • Tests sind die Spec
  • Kein Interpretationsspielraum mehr
  • Halluzinationen durch Tests aufgedeckt

Edge-Cases diktieren

  • Leerer Input / null / undefined
  • Unicode / Sonderzeichen
  • Sehr langer String / Grenzwerte
  • Duplikate / Reihenfolge-Varianten

TDD bei Bugs

  • "Schreib erst Test der Bug reproduziert"
  • Test ist rot → Bug bewiesen
  • Dann fixen bis Test grün
  • Test bleibt als Regression-Guard
💡 Trick: "Die Tests müssen jetzt rot sein — führe sie aus und zeig mir das Ergebnis" → verhindert Schummeln
🎯 Lernziel: Feature mit Tests vor Implementation gebaut, alle grün, kein Code selbst getippt
