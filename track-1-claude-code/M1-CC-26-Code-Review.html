4 Dimensionen

  • Correctness: Bugs finden
  • Reuse: Duplikation erkennen
  • Simplification: Komplexität reduzieren
  • Efficiency: Performance verbessern

Level-Guide

  • low: Pre-Commit-Check
  • medium: normale PRs
  • high: wichtige Features
  • ultra: Security-kritisch

Optionen

  • /code-review --fix auto-fixen
  • --comment als PR-Kommentare
  • /simplify vereinfachen

Standard-Workflow

  • add → /code-review
  • fixen → nochmal review
  • commit → /cost prüfen
