Konzept — Was ist ein Hook?
💡 Kerngedanke: Ein Hook ist ein Befehl, der automatisch läuft, wenn ein bestimmtes Ereignis eintritt. Du sagst es nicht jedes Mal — es passiert von selbst. Der klassische Fall: Jedes Mal, wenn Claude eine Datei schreibt, läuft danach Prettier und formatiert sie. Du musst nie wieder „bitte formatieren" tippen.

Der Unterschied zu einer Regel in CLAUDE.md: Eine Regel ist eine Bitte, die Claude befolgen kann. Ein Hook ist deterministisch — er läuft immer, ohne Ermessen. Für Formatieren und Testen willst du genau das.

Praxis — PostToolUse-Hook für Auto-Format

In .claude/settings.json kommt ein hooks-Block. Der matcher sagt, bei welchen Tools der Hook greift (hier Write|Edit), der command ist der Shell-Befehl:

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write . || true"
          }
        ]
      }
    ]
  }
}

Ein zweiter, nützlicher Hook: bei jedem Session-Start den git-Stand anzeigen:

{
  "hooks": {
    "SessionStart": [
      { "hooks": [ { "type": "command", "command": "git status --short" } ] }
    ]
  }
}
💡 Hooks können blockieren: Schlägt ein PreToolUse-Hook fehl (Exit-Code ungleich 0), stoppt er die Aktion. Bei PostToolUse-Format-Hooks willst du das nicht — darum hängst du || true an, damit ein Format-Fehler den Flow nicht lahmlegt.
Hook in Aktion

Szenario: Nach jedem Edit automatisch formatiert

1
Du bittest Claude, eine neue React-Komponente zu schreiben — er nutzt das Write-Tool
2
Der matcher Write|Edit greift → der PostToolUse-Hook startet automatisch
3
prettier --write formatiert die Datei: Einrückung, Anführungszeichen, Semikolons — sauber
4
Du hast nie „bitte formatieren" gesagt. Jede Datei ist ab jetzt automatisch konsistent
Der Gewinn
Formatierung ist kein Thema mehr. Kein Diff voller Whitespace-Änderungen, keine Stil-Diskussion im Team. Der Hook erledigt es deterministisch — bei jedem einzelnen Edit.
✅ Regel: Was immer passieren muss (Formatieren, Linten, Testen), gehört in einen Hook — nicht in CLAUDE.md. Was Ermessen braucht (Stil-Hinweise, Vorlieben), gehört in CLAUDE.md.
💡 Jetzt du: Lege in deiner .claude/settings.json einen PostToolUse-Hook mit matcher Write|Edit an, der npx prettier --write . || true ausführt. Starte eine neue Session, lass Claude eine Datei ändern und prüfe, ob sie danach automatisch formatiert ist.
