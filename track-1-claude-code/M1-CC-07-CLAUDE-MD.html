Konzept — Was ist CLAUDE.md

CLAUDE.md ist eine einfache Markdown-Datei im Projekt. Claude liest sie bei jedem Start automatisch — du musst sie nie erwähnen. Sie ist das Gedächtnis deines Projekts: Was ist das hier, wie wird gebaut, welche Regeln gelten. Ohne sie rät Claude bei jedem Befehl neu. Mit ihr arbeitet Claude vom ersten Prompt an wie ein eingearbeiteter Kollege.

💡 Faustregel: Alles, was du Claude in jeder Session erneut erklären müsstest, gehört in die CLAUDE.md. Einmal aufschreiben — für immer geladen.
Die 3 Ebenen

Global

  • ~/.claude/CLAUDE.md
  • Gilt für alle Projekte
  • Deine persönlichen Grundregeln
  • Sprache, Git-Identität, Stil

Projekt

  • ./CLAUDE.md im Root
  • Gilt nur für dieses Projekt
  • Wird ins Repo committed
  • Das Team teilt sie

Verschachtelt

  • ./backend/CLAUDE.md
  • Gilt nur im Unterordner
  • Für Monorepos / Teilbereiche
  • Ergänzt die Projekt-Datei
✅ Regel: Global = du. Projekt = das Repo. Verschachtelt = ein Teilbereich. Die spezifischere Datei ergänzt die allgemeinere — sie ersetzt sie nicht.

Echtes Beispiel — eine schlanke CLAUDE.md:

# CLAUDE.md

## Überblick
Landing-Page-Generator für RealRise. HTML/CSS/JS, kein Framework.

## Stack & Befehle
- Stack: reines HTML/CSS/JS, kein Build-Step
- Lokal testen: python3 -m http.server
- Deploy: ZIP nach Netlify ziehen

## Konventionen
- CSS-Variablen statt Hex-Werte (Design-Tokens)
- UI-Text auf Deutsch, Code/Kommentare auf Englisch
- Commit-Messages auf Deutsch

## Do / Don't
- NIE .claude/settings.local.json committen
- NIE inline-Styles — immer ins <style>
Praxis — /init nutzen

Du musst nicht bei null anfangen. Der Befehl /init scannt dein Projekt und schreibt einen ersten Entwurf:

# im Projektordner, in der Claude-Session
/init
# Claude liest package.json, Ordnerstruktur, README ...
# und schreibt eine CLAUDE.md mit ~70-100 Zeilen
💡 Jetzt du: Öffne dein Projekt, tippe /init, lies den Entwurf durch, streiche was falsch ist, ergänze deine Do/Don't-Regeln. Dann committen — siehe Szenario unten.

Ablauf: Eigene CLAUDE.md anlegen und committen

1
/init ausführen — Claude erzeugt den ersten Entwurf
2
Entwurf prüfen: Stack korrekt? Befehle echt? Unsinn raus
3
Do/Don't ergänzen — was Claude nie tun soll
4
Committen, damit das ganze Team dieselbe Basis hat
Die Befehle
git add CLAUDE.md
git commit -m "claude: projekt-gedächtnis angelegt"
Die häufigste Falle
🚫 Zu lang = teuer: Die CLAUDE.md wird in JEDEN Prompt geladen — jedes Token kostet. Eine 600-Zeilen-Datei verbrennt Geld und verwässert die wichtigen Regeln. Halte sie unter ~150 Zeilen.
⛔ Nur stabile Regeln: Keine Tagesnotizen, keine "heute machen wir X". Nur was dauerhaft gilt. Wechselndes Projektwissen gehört ins Memory-System (nächste Lektion).
