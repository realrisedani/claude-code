Wann Plan Mode

  • Neue Architektur entwerfen
  • Unbekannte Codebase erkunden
  • Riskante Migration vorbereiten
  • Großer Refactor (>5 Files)

Plan-Workflow

  • "Analysiere /src/auth" → Claude liest
  • "Erstelle Migrationsplan" → Claude plant
  • Plan reviewen → Freigeben
  • Execute-Mode → Claude implementiert

Plan committen

  • .claude/plans/ als Ordner anlegen
  • Datei: oauth-migration.md
  • Plan in Git → immer nachvollziehbar
  • Team kann Plan reviewen vor Execute

5-Punkte-Check

  • Files identifiziert?
  • Abhängigkeiten bekannt?
  • Rollback-Strategie vorhanden?
  • Tests im Plan enthalten?
  • Schritte atomar genug?
💡 Power-Move: Plan-Agent erstellt Plan → Execute-Agent implementiert → kein Kontext-Durcheinander zwischen Planung und Umsetzung
🎯 Lernziel: Plan Mode für echtes Feature, committed Plan, erster Schritt ohne Re-Work
