Default Mode

  • Fragt bei jeder Aktion nach
  • Sicher, aber etwas langsamer
  • Ideal für Produktiv-Code
  • Standard beim Start — immer safe

Accept-Edits

  • Schreibt ohne Rückfrage direkt
  • NUR mit sauberem git status!
  • Für Feature-Branches + Experimente
  • Niemals auf .env oder Auth-Files

Plan Mode (Shift+Tab)

  • Nur lesen, nie schreiben
  • Perfekt für Architektur-Sessions
  • Claude erklärt was er tun würde
  • Gut zum Review vor Accept-Edits

settings.json Allow/Deny

  • allow: ["Edit(src/**)","Bash(npm run *)"]
  • deny: ["Edit(.env*)","Bash(rm -rf*)"]
  • Committed ins Repo → Team-weit
  • Feinsteuerung ohne Modus-Wechsel
🚫 5 Rote Linien — NIE auf Accept-Edits: .env | Prod-DB-Config | Deploy-Scripts | Auth-Files | Shared Infra
