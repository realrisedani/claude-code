Warnzeichen erkennen

  • Vergisst frühere Anweisungen
  • Antwortet langsamer als sonst
  • Wiederholt sich oder widerspricht sich
  • Neue Fehler die vorher nicht da waren

Token-Verteilung

  • CLAUDE.md: ~5k Token (stabil)
  • Files: 20–80k Token
  • History: wächst mit jeder Runde
  • Tool-Outputs: können explodieren

Tool-Output-Disziplin

  • head -50 statt cat
  • jq '.data[:5]' statt jq '.'
  • ls statt find /
  • grep -n statt ganzes File lesen

Wann welche Strategie

  • Neues Thema → /clear
  • Erkenntnisse bewahren → state.md
  • Teilaufgabe isolieren → Sub-Agent
  • Context voll → Compaction auto
⚠ Anti-Pattern: cat auf große Datei = sofort 50.000+ Token verbraucht. Immer head/tail/grep nutzen!
