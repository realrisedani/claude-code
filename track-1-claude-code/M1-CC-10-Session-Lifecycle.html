Phasen

Lernziele

  • Den Lebenszyklus einer Session verstehen — Start, Arbeiten, Pause, Fortsetzen
  • Mit --resume und --continue nahtlos weiterarbeiten
  • Das state.md-Pattern nutzen, um den Stand vor jeder Pause festzuhalten
Konzept — Die 4 Phasen einer Session

Start

  • claude im Projektordner
  • CLAUDE.md wird geladen
  • Ziel der Session klären

Arbeiten

  • Bauen, testen, committen
  • Kontext sauber halten (/clear)
  • In Blöcken denken

Pause / Ende

  • Stand in state.md sichern
  • Committen, was fertig ist
  • Sauber abschließen
💡 Kerngedanke: Eine Session ist nichts Heiliges. Du darfst sie jederzeit beenden — solange du vorher festhältst, wo du stehst. Die vierte Phase ist das Fortsetzen: am nächsten Tag genau da weiter, wo du aufgehört hast.

Fortsetzen — resume und continue
# die zuletzt verwendete Session direkt weiterführen
claude --continue

# eine bestimmte ältere Session aus einer Liste auswählen
claude --resume
# → zeigt Liste vergangener Sessions zum Auswählen

⛔ Falle — blind auf resume verlassen: Die Session-History kann lang und verstopft sein. Für saubere Übergänge ist eine kurze state.md oft besser als die komplette alte Konversation wieder aufzuwärmen.
Das state.md-Pattern

Vor der Pause lässt du Claude den Stand in eine kleine Datei schreiben. Sie ist die Brücke zur nächsten Session — kurz, klar, sofort verständlich:

# state.md

## Stand
Landing-Page Hero + Features fertig. Pricing-Sektion fehlt noch.

## Als Nächstes
1. Pricing-Sektion mit 3 Karten bauen
2. Mobile-Ansicht der Features testen
3. Alles committen und nach Netlify deployen

## Offene Frage
Preis für mittleres Paket noch nicht final — bei Daniel nachfragen.
💡 Jetzt du: Lass Claude am Ende deiner nächsten Session sagen: "Schreib eine state.md mit Stand, nächsten Schritten und offenen Fragen." Beim nächsten Mal startest du mit: "Lies state.md und mach da weiter."
Übergang über Nacht — ein Beispiel

Szenario: Session unterbrochen, nächster Tag nahtlos weiter

1
Abends mitten in der Arbeit — du musst aufhören
2
Claude schreibt state.md mit Stand und nächsten Schritten
3
Fertiges committen, Rechner zu
4
Nächster Morgen: claude --continue, "Lies state.md" — sofort drin
Die Befehle
# abends
git add . && git commit -m "stand: hero + features fertig"
# morgens
claude --continue # "Lies state.md und mach weiter"
✅ Regel: Nie ohne Sicherung aufhören. Ein committeter Stand plus eine state.md kosten 30 Sekunden und retten den ganzen nächsten Tag.
