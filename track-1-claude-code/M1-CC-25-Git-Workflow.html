Conventional Format

  • feat(auth): add OAuth login
  • fix(api): handle null response
  • refactor(user): extract validation

Goldene Regeln

  • Atomar: 1 Commit = 1 Änderung
  • Max 72 Zeichen Subject-Zeile
  • Kein Punkt am Ende der Zeile

Branch-Strategie

  • main (geschützt, kein direkter Push)
  • develop → feature/name → PR
  • Review → Merge immer vollständig

CLAUDE.md Git-Regeln

  • NIE direkt auf main pushen
  • NIE --force-push verwenden
  • IMMER conventional format
⚠ Was Claude nie tun soll: "feat: update stuff" — nichtssagend. Immer: was+warum in 72 Zeichen. Gut: "fix(auth): redirect to login when session token expires"
🎯 Lernziel: Letzten 5 Commits in Conventional Format, saubere PR-Description geschrieben
