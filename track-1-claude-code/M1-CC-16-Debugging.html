Phase 1–2: Reproduce + Isolate

  • Reproduce: Bug zuverlässig triggern
  • Exakte Schritte dokumentieren
  • Isolate: minimaler Reproduce-Fall
  • Alles Unnötige entfernen

Phase 3–4: Hypothesize + Fix

  • Hypothesize: Ursache eingrenzen
  • Bisect: git bisect für Regression
  • Fix: minimale Änderung
  • Kein Rewrite — nur das Nötigste

Phase 5: Verify + Regress

  • Test schreiben der Bug bestätigt
  • Fix anwenden → Test grün
  • Regression-Test committen
  • Bug kommt nie wieder

Anti-Patterns

  • Fix ohne Reproduce versuchen
  • try-catch-Wrapper als Lösung
  • 50 console.logs ohne Plan
  • Symptom statt Ursache fixen
⚠ Goldene Regel: Kein Fix ohne Reproduce. Ein Bug der nicht reproduziert werden kann, kommt garantiert wieder.
🎯 Lernziel: 1 Bug durch alle 5 Phasen gelöst, committed Regression-Test
