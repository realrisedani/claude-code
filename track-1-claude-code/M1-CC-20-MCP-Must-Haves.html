GitHub + Filesystem

  • Issues / PRs / CI aus Claude
  • Code-Review ohne Browser
  • Files außerhalb Projekt lesen
  • Cross-Repo Analyse möglich

Notion + Linear

  • Pages / Databases schreiben
  • Knowledge Base updaten
  • Tickets / Sprints verwalten
  • Status-Updates automatisieren

Memory + Browser

  • Persistentes Wissen über Sessions
  • Kein Kontext-Verlust mehr
  • Headless-Browser steuern
  • UI-Tests automatisieren

Permissions-Disziplin

  • Schreibende MCP-Aktionen genehmigen
  • Lesen = OK, Schreiben = Review
  • Produktions-Daten extra sichern
  • Minimal Permissions Prinzip
💡 Empfehlung: Mit GitHub+Filesystem starten. Dann Notion wenn KB gebraucht. Browser für UI-Tests.
🎯 Lernziel: 2+ MCP-Server live, 1 echter Use-Case pro Server durchgespielt
