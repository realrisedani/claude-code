Konzept — Das Memory-System

Memory liegt im Ordner .claude/memory/. Es besteht aus einer MEMORY.md als Inhaltsverzeichnis und beliebig vielen Einzeldateien für einzelne Themen. Anders als die CLAUDE.md wird Memory nicht in jeden Prompt geladen, sondern nur wenn es zum Thema passt. So speicherst du beliebig viel Projektwissen, ohne dein Kontextfenster zu verstopfen.

💡 Bild dazu: CLAUDE.md ist das Merkblatt, das immer offen auf dem Tisch liegt. Memory ist das Aktenregal nebenan — Claude holt sich eine Akte nur, wenn er sie für die aktuelle Aufgabe braucht.

Der #-Shortcut

Du musst keine Datei von Hand öffnen. Tippe in der Session eine Zeile, die mit # beginnt — Claude schreibt sie automatisch ins passende Memory:

# direkt in der Claude-Session getippt:
# Der Apify-Token liegt in .env unter APIFY_TOKEN, nie hardcoden
# Deploy läuft über Netlify, ZIP wird per Drag&Drop hochgeladen
# → Claude legt/aktualisiert die Memory-Datei und meldet sich kurz
So sieht eine Memory-Datei aus
# MEMORY.md (der Index)

# Memory Index
- [Deploy-Workflow](deploy_workflow.md) — Netlify, ZIP, Drag&Drop
- [Kunde X Setup](kunde_x.md) — Stack, Zugänge-Hinweis, Ansprechpartner

# --- deploy_workflow.md (eine Einzeldatei)

# Deploy-Workflow
- Build: kein Step nötig (reines HTML/CSS/JS)
- ZIP erzeugen, in Netlify-Dashboard ziehen
- Live-URL danach im Kanal posten
Praxis — Eintrag anlegen, Index pflegen
💡 Jetzt du: Schreib mit # eine Tatsache über dein Projekt ins Memory (z. B. wie dein Deploy läuft). Dann bitte Claude: "Pflege das in MEMORY.md als Index-Eintrag ein." Schau dir den neuen Ordner .claude/memory/ an.

Ablauf: Vom Fakt zum gepflegten Memory

1
Mit # eine Tatsache notieren — Claude legt die Datei an
2
Claude bitten, den Eintrag in MEMORY.md zu verlinken
3
Index lesen — passt die Beschreibung, findet man es wieder?
4
Veraltetes alle paar Wochen löschen — Memory schlank halten
Optional committen (Team-Wissen)
git add .claude/memory/
git commit -m "memory: deploy-workflow festgehalten"
🚫 Keine Secrets ins Memory: Kein Token, kein Passwort, keine Kunden-Zugänge im Klartext. Schreib nur "Token liegt in .env" — nie den Wert selbst.
