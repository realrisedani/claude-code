Jede gute Claude-Session durchläuft diese vier Phasen. Das mentale Modell: Claude soll erst verstehen, dann planen, dann bauen und am Ende prüfen. Wer eine Phase überspringt, bekommt schlechtere Outputs — die Karten zeigen, was in jeder Phase passiert und was das Überspringen kostet.

READ — Verstehen

  • Claude liest relevante Files zuerst
  • Stellt Fragen, sammelt Kontext
  • Sag ihm welche: "Lies src/auth.ts"
  • Keine Änderungen
Ohne READ → falsche Annahmen, mehr Halluzination, falsche Files

PLAN — Planen

  • Claude erstellt einen Aktionsplan
  • Beschreibt Vorgehen, noch kein Code
  • Plan genehmigen vor dem Start
  • Korrekturen passieren hier
Ohne Plan → wilde Änderungen überall im Code

EXECUTE — Bauen

  • Plan genehmigt → implementieren
  • Edit/Write-Tools schreiben Dateien
  • Ändert Code aktiv
  • Erst nach Plan-OK
Zu früh gestartet → Arbeit auf falscher Grundlage

VERIFY — Prüfen

  • "Führe Tests aus" verlangen
  • Bash-Befehle wie npm test
  • Claude prüft eigene Arbeit
  • Bestätigt den Status
Kein Verify → ungetesteter, unbestätigter Code
⚠ Loop unterbrochen: Wird der Loop mittendrin abgebrochen, bleibt halbfertige Arbeit zurück. Lass Claude die vier Phasen vollständig durchlaufen, bevor du eine neue Aufgabe startest.
💡 Profi-Prompt: "Bevor du anfängst: 1) Welche Files liest du? 2) Was ist dein Plan? 3) Wie verifizierst du?" — das erzwingt alle 4 Phasen.

Plan Mode

💡 Profi-Tipp: Bei jeder größeren Aufgabe zuerst fragen: "Welche Files liest du? Was ist dein Plan? Wie verifizierst du?" — so behältst du die Kontrolle über den Loop.
Praxis-Übung

Aufgabe

Öffne Claude Code mit claude und gib eine komplexe Aufgabe ein — zum Beispiel "Baue eine Todo-App mit LocalStorage". Benutze dann /plan um sicherzustellen, dass Claude erst einen Plan erstellt bevor er implementiert. Beobachte welche der vier Phasen er durchläuft und wo du eingreifen müsstest, um eine übersprungene Phase nachzuholen.

Wissens-Check

3 Fragen zum Nachdenken

  1. Nenne die vier Phasen in der richtigen Reihenfolge — und was riskierst du, wenn Claude die READ-Phase überspringt?
  2. An welchen Signalen in der Ausgabe erkennst du, dass Claude sich in der EXECUTE-Phase befindet?
  3. Claude hat zu früh angefangen zu schreiben. Mit welchem Prompt oder Befehl steuerst du ihn zurück in die PLAN-Phase?
🎯 Lernziel erreicht: 4-Phasen-Loop verstanden, Phasen in der Ausgabe erkannt, aktive Steuerung per Prompt und /plan beherrscht
