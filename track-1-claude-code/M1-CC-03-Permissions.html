Lernziele

  • Die 3 Permission-Modi kennen und bewusst wählen
  • Permissions per .claude/settings.json festlegen und ins Repo committen
  • git-Disziplin als Schutz gegen Datenverlust beim Einsatz von Accept Edits verstehen
Konzept — Die 3 Permission-Modi

Default

  • Fragt bei jeder Aktion nach
  • Sicher, aber etwas langsamer
  • Ideal für Produktiv-Code
  • Standard beim Start — immer safe

Accept Edits

  • Schreibt ohne Rückfrage direkt
  • NUR mit sauberem git status!
  • Für Feature-Branches + Experimente
  • Niemals auf .env oder Auth-Files

Plan Mode (Shift+Tab)

  • Nur lesen, nie schreiben
  • Perfekt für Architektur-Sessions
  • Claude erklärt was er tun würde
  • Gut zum Review vor Accept Edits
💡 Feinsteuerung per settings.json: Mit allow (z. B. ["Edit(src/**)","Bash(npm run *)"]) und deny (z. B. ["Edit(.env*)","Bash(rm -rf*)"]) steuerst du Berechtigungen ohne Modus-Wechsel — committed gilt das team-weit.
🚫 5 Rote Linien — NIE auf Accept Edits: .env | Prod-DB-Config | Deploy-Scripts | Auth-Files | Shared Infra
Praxis — Permissions ins Projekt committen
💡 Warum: Ohne committed permissions muss jedes Teammitglied (oder du selbst nach einer Neuinstallation) alle Berechtigungen neu bestätigen. Einmal festgelegt — für alle gültig.

Datei anlegen: .claude/settings.json

{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(npm *)",
      "Read",
      "Write",
      "Edit"
    ]
  }
}
# settings.json committen (DIESE Datei ist sicher)
git add .claude/settings.json
git commit -m "claude: permissions für dieses projekt"
⛔ NIEMALS committen: .claude/settings.local.json — diese Datei enthält deine persönlichen Secrets und API-Tokens.
# settings.local.json dauerhaft aus git ausschließen
echo ".claude/settings.local.json" >> .gitignore
git add .gitignore && git commit -m "gitignore: claude local settings"

Warum Accept Edits ohne sauberen git GEFÄHRLICH ist

Szenario: Was passiert wenn git-Stand nicht sauber ist

1
Du hast 3 Stunden Arbeit ungesaved — Änderungen in 8 Dateien, kein Commit
2
Du aktivierst Accept Edits mit Shift+Tab
3
Claude überschreibt 5 dieser Dateien automatisch ohne zu fragen
4
Deine 3 Stunden Arbeit sind weg — kein Undo, kein git-Rollback möglich
Die Regel — vor jedem Accept Edits
git status # muss "nothing to commit" zeigen
git add .
git commit -m "checkpoint vor claude session"
✅ Goldene Regel: Sauberer git-Stand ist die einzige Absicherung gegen Datenverlust beim Einsatz von Accept Edits oder Auto Mode.
