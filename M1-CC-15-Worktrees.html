Wann Worktree

  • Großer Refactor geplant
  • Hotfix während Feature-Branch
  • Claude isoliert arbeiten lassen
  • Mehrere Features parallel

Setup-Befehl

  • git worktree add ../projekt-feat
  • Branch: feature/big-refactor
  • cd in Worktree → claude starten
  • Main bleibt unberührt

Sicherheitsnetz

  • Claude baut Mist → Worktree löschen
  • Kein Schaden an main
  • Neuer Worktree → frischer Start
  • Git History bleibt sauber

Aufräumen

  • git worktree list → Übersicht
  • git worktree remove [path]
  • git worktree prune → Alte bereinigen
  • Wöchentlicher Sweep einplanen
💡 Reflex: Refactor >1h Aufwand → immer Worktree. Kostet 30 Sek, spart Stunden Debugging.
🎯 Lernziel: Worktree als Reflex bei jedem größeren Refactor einsetzen
