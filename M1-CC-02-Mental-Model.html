READ-Phase

  • Claude liest relevante Files zuerst
  • Sag ihm welche: "Lies src/auth.ts"
  • Ohne READ → falsche Annahmen
  • Mehr READ = weniger Halluzination

PLAN-Phase

  • Claude erstellt Aktionsplan
  • "Erkläre erst dein Vorgehen" erzwingen
  • Plan genehmigen → dann erst starten
  • Ohne Plan → wilde Änderungen überall

EXECUTE + VERIFY

  • Plan genehmigt → implementieren lassen
  • "Führe Tests aus" danach verlangen
  • Verify = Claude prüft eigene Arbeit
  • Kein Verify → ungetesteter Code

Häufige Fehler

  • Überspringt READ → falsche Files
  • Kein Plan → wilde Änderungen
  • Kein Verify → ungetesteter Code
  • Loop unterbrochen → halbfertige Arbeit
💡 Profi-Prompt: "Bevor du anfängst: 1) Welche Files liest du? 2) Was ist dein Plan? 3) Wie verifizierst du?"
