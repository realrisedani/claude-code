Phase 1–2: Setup

  • git install — brew install git / winget
  • user.name — git config --global
  • user.email — git config --global
  • git --version — prüfen ob klappt

Phase 3–4: Erster Commit

  • git init — Repo initialisieren
  • README.md — erste Datei anlegen
  • git add . — alle Änderungen stagen
  • git commit -m "..." — speichern

Phase 5–6: GitHub

  • Account — github.com erstellen
  • Repo anlegen — New Repository
  • git remote add origin — verbinden
  • git push — hochladen

Tägliche 4 Befehle

  • git add . — Änderungen markieren
  • git commit -m "was" — speichern
  • git push — zur Cloud
  • git pull — von Cloud holen
⚠ main vs master: Ältere Git-Versionen nutzen "master", neuere "main". Bei Fehler: git branch -m master main
