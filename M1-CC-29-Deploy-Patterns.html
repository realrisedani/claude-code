Pre-Deploy 7 Punkte

  • Tests grün? TypeCheck?
  • .env Prod? Migration backwards-compat?
  • Rollback-Plan? Monitoring? Staging?

Rollback-Optionen

  • Vercel/Netlify: "Promote previous"
  • git revert | kubectl rollout undo
  • DB-Snapshot restore vorbereiten

Hotfix-Disziplin

  • Branch von main → minimaler Fix
  • Sofort deployen nach Review
  • Cherry-Pick auf develop danach

Feature-Flags

  • 1% → 10% → 100% Rollout
  • Kill-Switch sofort ohne Deploy
  • Kein Risiko bei neuem Feature
🚨 Nie ohne Rollback-Plan deployen. Produktion geht immer mal kaputt. 5-Min-Rollback rettet Nächte.
🎯 Lernziel: /predeploy-Command + Rollback geübt + Post-Mortem-Template bereit
