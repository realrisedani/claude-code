03 — Voice & Tone

Stimme & Ton

Authentisch, direkt, persönlich — nie corporate. Immer auf Deutsch für die DACH-Audience.

◆
Authentisch & Direkt
Keine Agentur-Sprache. Daniel spricht wie ein Freund der es wirklich weiß — nicht wie ein Berater der eine Pitch-Deck liest.
▲
Persönliche Brand
@daniel.realrise — nicht RealRise Agency GmbH. Ich-Perspektive, persönliche Erfahrungen, echte Resultate.
●
KI Educator
KI-Automation erklären wie man einem Freund erklärt warum Bitcoin 2013 interessant war. Einfach, konkret, mit FOMO-Einschlag.
▼
Urgency ohne Druck
Der Zug fährt ab — aber ich zeige dir die Tür. Nie manipulativ, immer wertbasiert. Die Dringlichkeit entsteht durch den Content selbst.
★
DM-First Strategie
Kein Link in der Bio-Chaos. Der Move ist: Reel → Keyword-DM → Funnel. Persönlich, skalierbar, konvertierend.
◇
DACH Fokus
Inhalte auf Deutsch. Begriffe wie „KI", „Automatisierung", „Reseller" — keine anglisierten Buzzwords ohne Erklärung.
DO — So klingt es richtig
  • ✓ „Ich nutze KI täglich — und jetzt kannst du es auch."
  • ✓ „Schreib mir KI per DM — ich schick dir alles was du brauchst."
  • ✓ „Das hier ist der einfachste Weg rein — ich zeig dir wie."
  • ✓ Erste Person, konkrete Zahlen, klare nächste Schritte
DON'T — Das vermeiden
  • ✗ „Unsere KI-Lösungen transformieren Ihren Workflow nachhaltig."
  • ✗ „Jetzt kostenlos registrieren und von exklusiven Benefits profitieren."
  • ✗ Passive Sprache, Corporate Speak, anglisierte Buzzwords ohne Kontext
  • ✗ Dritter Person, Wir-Sprache statt Ich-Perspektive

06 — CTA & Messaging

Call-to-Action

Ein CTA. Immer derselbe. Exakt dieser Wortlaut — keine Variationen.

„Schreib mir KI per DM —
ich schick dir alles was du brauchst"
EXAKTER WORTLAUT — NIEMALS ÄNDERN
Warum genau so?

Das Keyword „KI" triggert den ManyChat-Flow. Ohne das exakte Wort funktioniert die Automation nicht.

Platzierung

Am Ende jedes Reels (gesprochen + als Text-Overlay), auf jeder Funnel-Page als primärer Button/CTA.

Tone

Hilfreich, nicht verkäuferisch. „Ich schick dir" = persönliches Versprechen, kein generischer Button-Text.

07 — Critical Rules

11 Fehler die du vermeidest

Diese Regeln existieren aus Erfahrung. Jede davon wurde durch einen echten Fehler gelernt.

01
NIEMALS Unterordner in 04_Daniel_Pages/ verwenden — Netlify ZIP bricht bei Subfolders. Alles flat in einem Verzeichnis.
02
NIEMALS die GHL Form ID ändern. Sie ist hardcoded in jedem Funnel eingebettet. Änderungen brechen alle bestehenden Formulare.
03
NIEMALS settings.local.json committen. Die Datei enthält API-Tokens und ist immer in .gitignore einzutragen.
04
NIEMALS die Meta Pixel ID ändern: 1343968131131688 — alle Tracking-Daten und Custom Audiences sind an diese ID gebunden.
05
IMMER Flat-Struktur verwenden: /thema.html — nicht /thema/index.html. Netlify ZIP verarbeitet nur flache Strukturen korrekt.
06
IMMER _redirects erweitern bei neuen Themen: /thema /thema.html 200 — sonst 404 auf Clean URLs.
07
IMMER korrekten Affiliate-Parameter verwenden: ?source=daniel oder ?source=regina — nie andere Werte, nie weglassen.
08
IMMER die 3 exakten Schriften verwenden: Cormorant Garamond (Headlines), DM Mono (Labels/Code), DM Sans (Body). Kein Inter, kein Roboto.
09
Der CTA muss immer exakt diesen Wortlaut haben: „Schreib mir KI per DM — ich schick dir alles was du brauchst". Keine Variationen, keine Abkürzungen.
10
Deployment via ZIP drag&drop auf Netlify — nicht über „Browse Files". Der Browse-Files-Upload erzeugt keine korrekten Redirects.
11
Spec-Driven: Master Prompt und Template immer lesen bevor etwas gebaut wird. Niemals blind implementieren — immer Kontext verstehen.

09 — Deploy Workflow

Deploy-Prozess

Schritt für Schritt — jedes Mal gleich. Kein Abweichen vom Prozess.

1
HTML-Datei erstellen
Neue Datei in 04_Daniel_Pages/ anlegen. Name = Thema (lowercase, kebab-case). Kein Unterordner.
2
_redirects aktualisieren
Neuen Redirect-Eintrag hinzufügen. Format: /thema /thema.html 200
3
ZIP-Archiv erstellen
Den gesamten Deploy-Ordner als ZIP komprimieren. Alle Dateien müssen auf der obersten Ebene des ZIPs liegen.
4
Netlify — ZIP drag&drop
Auf Netlify Site öffnen → Deploys Tab → ZIP-Datei in den Drop-Bereich ziehen. NICHT „Browse Files" klicken.
✓
Verify & Test
Clean URL testen (/thema), Form-Submission prüfen, Meta Pixel im Browser-Dev-Tools bestätigen.
Nach dem Deploy testen
# Clean URL (nicht .html) ki.realrise-agency.com/daniel/thema # Form-Parameter prüfen ?source=daniel ← muss in URL sein # Pixel Debugger Facebook Events Manager → Verify Events → URL eingeben
Häufige Fehler
  • ! „Browse Files" statt drag&drop → Redirects funktionieren nicht
  • ! Unterordner im ZIP → 404 auf allen Seiten
  • ! _redirects vergessen → Clean URL gibt 404
  • ! ?source= fehlt → Affiliate-Tracking bricht
Git nach Deploy
git add 04_Daniel_Pages/thema.html git add _redirects git commit -m "neue Funnel-Page: [thema]" git push
